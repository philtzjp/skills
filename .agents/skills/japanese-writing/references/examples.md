# 場面別 before / after 例

## エージェントの応答（ユーザーへの報告）

### 例 1: 作業完了の報告

```
❌ バグの修正。テストの実行。問題なし。
✅ バグを修正し、テストを実行しました。問題はありませんでした。
```

### 例 2: 次のステップ

```
❌ 次は PR のマージまで。
✅ 次は PR をマージします。
```

### 例 3: 依頼

```
❌ ログの共有どうぞ。
✅ ログを共有してください。 / ログの共有をお願いします。
```

## コミットメッセージ

```
❌ 認証まわりの修正。
✅ fix: 認証エラーのハンドリングを修正

❌ 不要な依存関係を落とす
✅ chore: 未使用の依存関係を削除
```

## PR 説明

```
❌ この PR では変更を寄せます。レビューどうぞ。
✅ この PR で変更を取り込みます。レビューをお願いします。
```

```
❌ 既知の問題を倒す対応。
✅ 既知の問題を解消しました。
```

## 技術ドキュメント

```
❌ 設定ファイルを更新することができます。
✅ 設定ファイルを更新できます。
```

```
❌ デプロイの実施まで。
✅ デプロイは別途実施予定です。
```

## UI 文言

```
❌ 保存。
✅ 保存する

❌ エラー。再試行。
✅ エラーが発生しました。再試行してください。
```

## 長文の分割例

```
❌ 今回の変更では認証フローを更新し、トークンの検証ロジックを修正し、
   エラーハンドリングを改善し、テストを追加することができます。
✅ 今回の変更では、認証フローを更新し、トークン検証とエラーハンドリングを
   修正しました。あわせてテストを追加しています。
```

## 「〜きる」の修正例

```
❌ 仕様を書ききる
✅ 仕様を書き上げる / 仕様の起草を完了する

❌ タスクをやりきる
✅ タスクを完了する

❌ 一覧を読みきる
✅ 一覧を最後まで確認する
```
